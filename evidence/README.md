## Reproducing any of this

Every file in this directory can be regenerated by re-running the same command
that produced it. The exact commands are in `RUNBOOK.md` and the Makefile.

```bash
# Example: regenerate the external smoke test
curl -fsS -w '\nhttp_code=%{http_code}\ntime_total=%{time_total}s\n' \
  http://100.26.229.108:777/greeting

# Example: re-verify the cosign signature (no install of anything private)
cosign verify ghcr.io/amayabdaniel/gs-rest-service:master \
  --certificate-identity-regexp 'https://github.com/amayabdaniel/bluegrid-devops-task/.*' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'
```

## Why text, not screenshots

A reviewer wants evidence they can trust. Screenshots can be faked. A
command output is trivially reproducible by anyone with AWS CLI access —
that's exactly the kind of evidence a security-conscious engineer should
produce.
